# ADR-005：未知结果恢复

## 决定

操作具有 `pending`、`succeeded`、`failed`、`unknown` 四个持久状态。同步业务在提交事务内形成终态；支付和 Mint 允许由外部事实推进。随机结果在第一次裁决时持久化，不得重新生成。

前端在生成并保存 UUID 后立即进入 `confirming`，下一帧进入 `submitting`，但不提前宣告业务成功。网络中断只进入 `unknown`，仅锁定同一 `use_case` 的再次提交，主导航继续可用。恢复只查询原操作。`me/bootstrap` 返回阻塞操作、待处理支付和 Mint，以便重新进入后继续恢复。

终态操作保留 30 天；非终态操作不得清理。
