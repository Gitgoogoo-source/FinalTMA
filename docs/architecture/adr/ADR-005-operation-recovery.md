# ADR-005：未知结果恢复

## 决定

操作具有 `pending`、`succeeded`、`failed`、`unknown` 四个持久状态。同步业务在提交事务内形成终态；支付和 Mint 允许由外部事实推进。随机结果在第一次裁决时持久化，不得重新生成。

前端在生成并保存 UUID 后立即进入 `confirming`，下一帧进入 `submitting`，但不提前宣告业务成功。网络中断只进入 `unknown`，恢复只查询原操作。默认只锁定同一 `use_case` 的再次提交；`gacha.open` 与 `wheel.spin` 从 `confirming` 开始禁用本领域操作按钮和五个底部导航，经过 `submitting`、`pending`、`unknown` 持续锁定，并在服务端成功或失败结果弹窗中保持锁定，直到用户通过弹窗规定动作处理该结果。弹窗内的结果处理和恢复按钮不被这把锁禁用。`me/bootstrap` 返回阻塞操作、待处理支付和 Mint，以便重新进入后继续恢复。

开盒和转盘终态使用服务端 `result_acknowledged_at` 回执。未确认的成功与失败结果和非终态一起在重新进入时恢复；各自的专用轻量查询负责发现晚于首屏事务快照提交的原操作。用户处理结果前，专用 RPC 按当前身份、原用例与终态条件原子确认；确认前持续恢复原完整 payload，确认后不再展示。浏览器持久存储不保存或裁决随机结果事实。

操作注册中心只对 `succeeded` 且通过原命令输出 Schema 校验的结果开放业务成功视图。需要让用户核对最终业务事实的命令使用领域专用结果组件；开盒与转盘结果组件只消费各自服务端结果，并按照产品主文档展示单次或十次结果。其余操作继续使用通用状态弹窗，`pending`、`unknown`、`failed` 或 Schema 不完整的结果不得进入成功视图；持久成功结果的 Schema 不完整时回到 `unknown` 查询原操作，在完整结果恢复前禁止确认。

已确认展示的终态操作保留 30 天；非终态操作与未确认展示的开盒、转盘终态不得清理。
