# ADR-009：开盒页运行期视图状态

## 决定

开盒页的档次与页面滚动位置只保存在 Web 运行内存中，以当前已确认会话的 `userId` 隔离。会话敏感状态清理时清空全部开盒页记忆并递增失效代次，失效代次之前卸载的页面不得把旧状态重新写回。Mini App 重新启动不会恢复这份内存状态，也不使用 `localStorage`、`sessionStorage` 或服务端数据保存视图记忆。

开盒页挂载时先以充值恢复 query 参数中的有效档次覆盖当前用户的普通页面记忆，否则恢复记忆档次，并在首帧可用布局中恢复滚动位置。页面同时强制重新读取 `gacha.bootstrap`；只有本次挂载后的服务端读取确认存在免费稀有资格时，才将最终选中档次回正为稀有盲盒。充值恢复的单抽或十连意图只显示再次确认入口，不触发开盒请求。

价格、概率、保底、免费资格和开盒结果始终来自服务端响应。运行期视图状态不参与资产、资格、扣款、随机结果或业务成功判断。

结果弹窗使用服务端已确认的付费方式、最终稀有度、保底目标和固定保底触发标记解释保底变化。付费结果达到目标且未标记固定保底触发时，界面明确标记为自然命中重置；免费结果不生成任何重置说明。该说明只解释服务端已经裁决的结果，不参与保底累计、重置或最终进度计算。
