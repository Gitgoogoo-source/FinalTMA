# ADR-011：Monster Tamer 登录后嵌入式游戏

- 状态：已接受
- 日期：2026-07-23
- 取代：本 ADR 的旧版“公开独立静态子应用”定义
- 依据：[Monster Tamer 玩法说明](../../../monster玩法说明.md)与产品第 21 章

## 背景

Monster Tamer 必须使用当前玩家在 TMA 中真实可用的藏品出战，并保存跨设备探索进度。旧版公开静态页、本地怪兽、本地存档、捕捉和经验体系不能验证登录身份、藏品可用性或并发占用状态，已经被产品裁决替代。

游戏仍位于“游戏”页首位，藏品远征和幸运转盘保持原顺序与各自状态。玩家点击入口后应留在已经登录的 React 应用中，以全屏覆盖层运行；关闭覆盖层后直接回到原页面。

## 决策

### 运行边界

1. `apps/web/src/domains/monster-tamer` 拥有入口、全屏覆盖层、准备页、营地、队伍、HUD、小地图、错误状态和 Phaser 挂载生命周期。
2. Phaser 作为 Web bundle 的延迟加载运行时挂载到覆盖层容器，只负责地图、移动、碰撞、可见敌人、相机和战斗表现。
3. React 负责认证后的 API 调用。Phaser 永远不接收 access token、session、用户 ID、Supabase client 或 API client，只能通过带类型的 bridge 提交动作意图并接收服务端权威快照。
4. `/monster-tamer` 与 `/monster-tamer/` 进入主应用 SPA，并在通过统一账号门禁后跳转到 `/game`；仓库删除旧 `apps/web/public/monster-tamer` 整包，不再发布独立 HTML、旧脚本、旧数据或旧素材。
5. 覆盖层关闭、会话 generation 变化、账号封禁或组件卸载时必须销毁 Phaser、监听器、动态纹理和音频，不能让旧会话快照继续运行。

### 数据与可信边界

1. `monster_tamer.bootstrap`（`GET /api/monster-tamer/bootstrap`）返回服务端确认的可用藏品、210 份确定战斗资料、世界定义、进度和活动战斗。
2. 背包与队伍只允许 `inventory.available_quantity(...) > 0` 的模板；确认队伍、进入区域和开始战斗均在事务内重新校验。
3. `monster_tamer.checkpoint`（`POST /api/monster-tamer/checkpoint`）只接受队伍确认、区域进入、世界节点完成、使用内部补给，以及带顺序的经过格与迷雾格同步意图。服务端从已保存位置逐格验证相邻关系、固定可通行格和能力门禁；节点、出口和区域进入只按验证后的权威位置裁决。
4. `monster_tamer.battle`（`POST /api/monster-tamer/battle`）只接受从权威位置开始固定遭遇、从已解锁祭坛开始首领再战，或在活动战斗中使用一个技能的意图；遭遇半径、路线、生命、攻击、附加效果、敌方回合、胜负、内部补给、首领能力和通关全部由数据库 RPC 裁决。
5. 每个写接口使用 `Idempotency-Key`、请求摘要、操作记录、行锁、进度/战斗版本和同一事务。重复相同请求返回同一结果；版本冲突要求采用最新服务端快照。
6. `monster_tamer` schema 只保存本游戏进度和战斗。它可以读取正式目录及藏品可用量，不得写入 holdings、reservations、余额、账本、远征、市场、进化、分解、Mint、任务、邀请、VIP、支付或链上状态。
7. 不创建浏览器本地业务存档，不使用 `MONSTER_TAMER_DATA`、`localStorage`、`sessionStorage`、Cookie 或 IndexedDB 保存游戏进度。
8. 服务端进度保存当前区域和权威 64px 网格坐标。响应未知时，Phaser 命令 UUID 继续作为原 `operation_id`；前端只查询该操作，只有明确 `OPERATION_NOT_FOUND` 时才能用同一 UUID、同一路由和同一输入单次重提。

### 世界与战斗

1. 世界由中心营地、萤光森林、潮汐湿地、风蚀高原、晶矿洞窟、熔火盆地、隐藏洞穴和最终守护者巢穴组成，采用 64px 网格的八张独立 Tilemap。
2. 普通怪、精英、五个区域首领和最终守护者均来自正式 210 模板目录；世界中显示目录缩略图，战斗中按需加载详情图。普通怪和精英只在服务端固定接战半径内表现警觉与追逐，首领和最终守护者保持固定。
3. 普通怪和采集点随区域重入刷新；精英、宝箱、永久门禁、区域首领首次胜利、五种能力和最终通关由服务端永久记录。七个野外区域各有固定返回营地出口；五个区域首领和最终守护者首次胜利后移除原实体，并永久开放对应再战祭坛。
4. 五种属性只影响技能附加效果、区域环境反馈和视觉效果，不提供固定伤害倍率。
5. 每个模板始终有三个技能；生命、攻击和技能强度只由 `combat_power`、稀有度、阶段、进化链属性和固定技能资料确定。
6. 游戏不存在捕捉、本地怪兽、等级、经验、教学剧情、任务链、解谜、昼夜或天气系统。

## 结果

登录、藏品归属与可用量、进度、首领能力和战斗结果均有服务端唯一事实来源；同一账号可以跨设备继续。Phaser 故障不能取得业务写权限，前端篡改只能产生被 RPC 拒绝的动作意图。

代价是游戏需要认证 API、数据库状态机和严格的客户端生命周期管理。发布门禁必须同时验证路由契约、Schema/RPC、安全授权、React/Phaser bridge、无浏览器存档、旧公开入口不可达以及游戏页组合顺序。
