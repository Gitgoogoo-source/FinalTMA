# 回滚

## 数据库迁移前

静态门禁、素材门禁、账号配置或部署预检失败时停止发布，不修改 Telegram 入口和 Cron。

## 数据库迁移后、应用切换前

不执行向后迁移，不恢复旧 schema。保持 Telegram 入口关闭，修复同一重构基线并以新的前向迁移继续；重新在真实开发环境完成验收后再进入生产。

## 应用切换后

只有目标 commit 与当前迁移序列兼容时才能回滚应用。不存在兼容 commit 时关闭 Telegram 入口和 Cron，保留 Functions 供支付/Mint 恢复操作，使用前向修复发布。

Stars 已确认付款、已交付资产与 TON 链上事实不可通过代码回滚撤销。支付和 Mint 必须通过原订单、原操作和对账任务完成，不得创建替代订单、重复发放或伪造链上状态。
