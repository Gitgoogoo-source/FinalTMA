# Monster Tamer 玩法说明

## 1. 文档状态

本文记录 Monster Tamer 当前唯一正确的玩法方案，只保留已经确认的最终结论。

本文替代现有资料中关于“公开独立静态游戏”“本地怪兽目录”“怪兽捕捉”“本地等级经验”“本地单机存档”和“公开路径直接游玩”的旧定义。Monster Tamer 的后续产品、架构、接口、数据库、地图、战斗和界面实现均以本文为准。

## 2. 游戏定位

Monster Tamer 是嵌入 FinalTMA 的登录后全屏怪兽探索游戏，不是公开独立子应用。

玩家在 FinalTMA 游戏页点击 Monster Tamer 后，游戏以全屏覆盖层打开，继续使用当前 TMA 内存会话。关闭游戏后回到原游戏页，远征和幸运转盘的页面状态保持不变。

无法验证登录身份、会话已经失效或账号状态不允许进入时，不加载 Monster Tamer 地图，返回 FinalTMA 的正常登录与账号门禁流程。直接访问旧的公开 Monster Tamer 路径不能绕过登录进入游戏。

Monster Tamer 的玩家怪兽全部来自该玩家在 TMA 中真实可用的藏品。游戏没有自有玩家怪兽目录，不提供临时怪兽，不允许捕捉怪兽，也不会发放 TMA 藏品。

## 3. 核心体验

游戏采用明亮自然奇幻的精细顶视角像素风，继续使用 Phaser Tilemap 和 64px 世界网格。

首次完成主要探索的目标时长为 90–120 分钟。核心循环固定为：

1. 在中心营地读取真实可用藏品。
2. 从背包选择 1–3 个不同藏品模板组成队伍。
3. 进入已经开放的生态区域。
4. 探索地图、发现道路、补给点、宝箱、精英怪和区域首领。
5. 主动接触地图上可见的生态怪并进行回合制战斗。
6. 击败区域首领，永久获得对应探索能力。
7. 使用探索能力直接通过新的地形门禁、打开捷径并进入后续区域。
8. 击败五个区域首领后进入最终守护者巢穴。
9. 击败最终守护者后记录通关，继续保留自由探索、隐藏内容和首领再战。

游戏不设置强制剧情、教学剧情、任务列表、委托、连续 NPC 支线或解谜玩法。玩家依靠地形、道路、可见敌人、宝箱、区域地标和探索能力自主决定行动顺序。

游戏不设置昼夜和天气系统。区域环境、敌人分布、道路与能力门禁保持确定，不要求玩家等待现实时间或游戏时间。

## 4. 世界地图

世界固定包含：

- 1 个中心营地。
- 5 个主要生态区域。
- 1 个隐藏洞穴。
- 1 个最终守护者巢穴。

### 4.1 中心营地

中心营地是每次探索的准备与恢复地点，固定承担以下功能：

- 读取并展示当前真实可用的 TMA 藏品。
- 选择和更换 1–3 个出战藏品。
- 免费完全恢复队伍。
- 查看已经获得的探索能力。
- 查看已发现区域和地图完成状态。
- 进入当前已经开放的生态区域。
- 在藏品不足时返回 TMA 藏品页或盲盒页。

玩家只能在中心营地更换队伍。进入生态区域后不能临时替换队员。

### 4.2 半开放地图结构

开局开放萤光森林和潮汐湿地，玩家自行决定先探索哪一个区域。

固定开放关系如下：

```text
中心营地
├── 萤光森林 ── 击败首领 ── 风蚀高原
└── 潮汐湿地 ── 击败首领 ── 晶矿洞窟

风蚀高原首领 + 晶矿洞窟首领
└── 熔火盆地

熔火盆地首领
└── 最终守护者巢穴
```

隐藏洞穴属于非主路线探索区域，通过已经获得的区域能力直接打开入口。隐藏洞穴不包含解谜流程。

### 4.3 五个区域与探索能力

| 区域 | 区域首领胜利后获得的永久能力 | 能力用途 |
| --- | --- | --- |
| 萤光森林 | 藤桥生长 | 让枯藤形成桥梁并清除荆棘封锁 |
| 潮汐湿地 | 潮汐行走 | 穿过深水和激流并启动水流通路 |
| 风蚀高原 | 风流滑翔 | 利用上升气流跨越断崖 |
| 晶矿洞窟 | 雷能充能 | 启动遗迹机械、升降台和封闭矿门 |
| 熔火盆地 | 炽热护盾 | 穿过熔岩热浪并开启最终守护者巢穴 |

探索能力属于 Monster Tamer 服务端进度，不属于 TMA 藏品属性。能力一经获得永久生效，不要求玩家携带特定藏品。

所有能力门禁采用直接、明确的交互：玩家拥有能力时立即通过，没有能力时显示缺少的能力。能力门禁不附带推箱、线路排列、开关顺序、错误重置或多步骤谜题。

## 5. TMA 藏品读取

### 5.1 唯一玩家怪兽来源

Monster Tamer 只通过同源服务端接口读取当前登录玩家的 TMA 藏品，不允许前端直连 Supabase，也不接受前端提交的藏品归属、数量、战斗力、稀有度、阶段或图片数据作为真实结果。

背包只展示服务端确认满足以下条件的藏品模板：

```text
available_quantity > 0
```

被出售、远征、交易或 Mint 占用而导致可用数量为 `0` 的模板不出现在 Monster Tamer 背包中。

每张背包卡固定展示：

- 正式模板图片。
- 正式名称。
- 水、火、木、风、雷中的固定属性。
- 稀有度。
- 进化阶段。
- `combat_power`。
- 当前 `available_quantity`。
- 是否已经进入当前队伍。

### 5.2 无可用藏品

玩家没有任何 `available_quantity > 0` 的藏品时，停留在游戏准备页，不加载世界地图，不提供临时怪兽，也不允许触发战斗。

准备页固定显示“暂无可出战藏品”，并提供返回 TMA 藏品页和盲盒页的入口。

### 5.3 队伍规则

- 队伍最少 1 个、最多 3 个藏品模板。
- 同一模板不能重复进入队伍。
- 藏品可用数量大于 `0` 即具备一次选入资格；重复数量不会产生重复队员。
- 队伍只在中心营地更换。
- 玩家可以使用与敌方生态怪、区域首领或最终守护者相同的藏品模板。

### 5.4 可用性重新校验

进入生态区域前和每场战斗开始前，服务端重新校验全部出战模板的 `available_quantity`。

任一队员不再可用时：

1. 不开始区域切换或战斗。
2. 返回中心营地。
3. 重新读取背包。
4. 清除无效队伍结果。
5. 要求玩家重新组队。

游戏不能沿用旧快照继续使用已经失去可用资格的藏品。

## 6. 藏品战斗资料

### 6.1 战斗数值来源

Monster Tamer 不设置独立等级和经验。

玩家和敌人的生命、攻击与技能强度由以下 TMA 正式数据确定：

- `combat_power`。
- 稀有度。
- 进化阶段。
- 所属进化链的固定生态属性。
- 该模板所属阶段的固定技能效果。

战斗不会改变 TMA 藏品的 `combat_power`、稀有度、阶段、数量、归属或任何资产状态。

### 6.2 五种生态属性

生态属性固定为：

- 水。
- 火。
- 木。
- 风。
- 雷。

70 条进化链各自固定一种属性，同一进化链的三个阶段保持同一属性，共形成 210 份确定的战斗资料。

不设置战斗定位，不把藏品划分为攻击、防御、辅助或敏捷职业。

属性之间不产生固定伤害克制，不存在属性伤害倍率。生态属性只影响：

- 技能附加效果。
- 区域环境效果。
- 技能动画和视觉反馈。

### 6.3 固定三个技能

每个藏品模板固定拥有 3 个技能。

同一进化链的三个阶段保持一致的技能主题和按钮数量。高阶段只强化技能数值、持续时间或附加效果，不增加技能数量，不要求玩家学习、替换或配置技能。

技能资料由服务端根据正式模板返回，前端不能自行决定技能、数值或效果。

## 7. 战斗系统

游戏保留现有回合制战斗框架，并增加：

- 区域首领专属战斗机制。
- 地图可见的精英怪。
- 固定区域环境效果。
- 首领再战。
- 基于当前队伍战斗力的柔性难度匹配。

不全面重写为实时战斗，也不加入怪兽捕捉流程。

### 7.1 敌方怪兽来源

普通生态怪、精英怪、五个区域首领和最终守护者全部来自 TMA 正式 210 模板目录。

敌方模板不受玩家是否拥有该模板影响。玩家可以遇到尚未拥有的藏品，但击败敌人不会获得、解锁或增加该 TMA 藏品。

五个区域首领分别使用对应生态属性的高阶正式模板。最终守护者使用固定的神话三阶正式模板。

### 7.2 地图遭遇

普通生态怪和精英怪直接显示在世界地图上，玩家可以观察、绕开或主动接触，不使用随机草地暗雷。

地图敌人使用藏品缩略图显示为悬浮生态头像，并具有：

- 对应属性光环。
- 待机漂浮。
- 发现玩家时的警觉反馈。
- 进入追逐状态的移动反馈。

进入战斗后使用该模板的详情图。

### 7.3 柔性难度

每个生态区域具有固定难度范围。敌人强度在该范围内根据当前队伍平均 `combat_power` 进行有限调整。

柔性调整不能消除高战斗力藏品的优势，也不能让低战斗力队伍完全无法进入前期区域。区域顺序、精英怪、区域首领和最终守护者继续保持明确的难度增长。

### 7.4 普通战斗收益

普通生态战斗和精英战斗只产生 Monster Tamer 内部结果，包括：

- 区域内治疗补给。
- 区域内临时增益。
- 通往宝箱、补给点、捷径或隐藏区域的道路清理结果。

战斗不产生 K-coin、Fgems、TMA 藏品、免费盲盒资格、任务进度、邀请进度、VIP 权益、市场资产、Mint 结果或链上资产。

### 7.5 恢复与失败

中心营地免费完全恢复队伍。

区域内固定通过以下内容恢复：

- 生态怪战斗掉落的治疗补给。
- 地图治疗果。
- 固定补给点。

战斗结束后不会自动完全恢复。

全队倒下后：

1. 返回中心营地。
2. 完全恢复队伍。
3. 保留已经捕获以外的全部合法游戏进度；游戏不存在捕获数据。
4. 保留已获得探索能力、已开启宝箱、已完成机关、已击败精英、首领胜利和最终通关记录。
5. 不扣减或改变任何 TMA 资产。

## 8. 刷新与重复游玩

- 普通生态怪在玩家离开区域并重新进入时刷新。
- 地图采集点在区域重入时刷新。
- 精英怪只记录首次击败结果，不随区域重入恢复首次奖励。
- 宝箱只开启一次。
- 能力门禁和已打开捷径永久记录。
- 区域首领首次击败后通过独立再战入口重复挑战。
- 最终守护者首次击败后继续提供再战。
- 再战不重复发放探索能力或首次通关状态。

## 9. 最终完成目标

玩家必须击败五个区域首领并获得全部五种探索能力，随后开启最终守护者巢穴。

首次击败最终守护者后，服务端记录通关状态。通关不会结束或清空存档，玩家继续保留：

- 自由探索。
- 地图迷雾完成度。
- 隐藏洞穴。
- 未开启宝箱。
- 未击败精英怪。
- 区域首领再战。
- 最终守护者再战。

通关状态只属于 Monster Tamer，不发放或改变 TMA 资产。

## 10. 界面

### 10.1 全屏游戏层

Monster Tamer 作为 FinalTMA React 应用中的全屏游戏层运行，不使用 iframe，不打开公开独立页面，不丢失当前内存会话。

Phaser 负责世界、角色、敌人、地图、相机、动画和战斗表现。React DOM 负责背包、队伍选择、HUD、小地图、错误状态和返回操作。

任何背包、地图或确认面板打开时，世界移动输入立即暂停，不能在面板下继续移动或触发战斗。

### 10.2 低占用 HUD

正常探索时固定显示：

- 左上：最多 3 个队员的头像和当前生命值。
- 右上：当前区域名称和已获得探索能力入口。
- 画面安全区边缘：移动端操作按钮和当前交互提示。

小地图、背包和详细资料通过按钮展开。画面中央和下方主要行动区域不放置常驻大面板。

### 10.3 迷雾地图

小地图只显示玩家已经走过的区域。

以下地点在发现后永久记录图标：

- 区域出口。
- 补给点。
- 宝箱。
- 能力门禁。
- 精英怪地点。
- 区域首领入口。

隐藏洞穴在玩家实际发现入口前不显示。

## 11. 服务端进度

Monster Tamer 允许写入自身独立的服务端游戏进度，实现同一 TMA 账号跨设备继续。

服务端进度固定包含：

- 当前中心营地或区域节点。
- 当前有效队伍模板 ID。
- 已开放区域。
- 已获得探索能力。
- 已揭开的地图迷雾。
- 已开启宝箱。
- 已完成能力门禁和捷径。
- 已首次击败的精英怪。
- 已首次击败的区域首领。
- 最终守护者通关状态。
- 最近一次自动保存节点。

地图中的瞬时动画、Phaser 对象、临时粒子和渲染状态不进入服务端存档。

### 11.1 自动保存节点

以下事件完成后自动保存：

- 在中心营地确认队伍。
- 进入或离开生态区域。
- 开启宝箱。
- 完成能力门禁或打开永久捷径。
- 首次击败精英怪。
- 首次击败区域首领。
- 获得探索能力。
- 首次击败最终守护者。

异常退出后从最近一次成功保存的节点恢复。

### 11.2 写入边界

Monster Tamer 只写入自身游戏进度，不写入或改变：

- `inventory.holdings`。
- `inventory.reservations`。
- 藏品可用数量。
- 远征状态。
- 市场挂单或交易。
- 进化状态。
- 分解结果。
- Mint 状态。
- K-coin 或 Fgems。
- 账本、支付、退款、任务、邀请或 VIP。

所有服务端进度写入必须经过已验证会话、稳定幂等键、具名数据库 RPC 和事务处理。数据库是 Monster Tamer 进度与藏品可用性的最终事实来源。

## 12. 错误与状态回正

| 场景 | 固定结果 |
| --- | --- |
| 会话无效或账号门禁失败 | 不加载游戏，返回 FinalTMA 登录与账号门禁 |
| 藏品读取中 | 准备页显示加载状态，不展示本地默认藏品 |
| 藏品读取失败 | 显示读取失败和重试；不能使用旧数据开始新区域或战斗 |
| 没有可用藏品 | 停留准备页，显示返回藏品页和盲盒页入口 |
| 队员在进入区域前失效 | 返回营地，刷新背包并重新组队 |
| 队员在战斗前失效 | 不开始战斗，返回营地，刷新背包并重新组队 |
| 游戏进度保存失败 | 保留当前可见状态并显示保存失败；不伪造已保存结果 |
| 重复保存请求 | 返回同一幂等结果，不重复推进首领、宝箱、能力或通关状态 |
| 目录图片加载失败 | 显示资源缺失状态，不替换为另一藏品图片 |

## 13. 当前图片使用规则

当前版本直接使用 TMA 正式目录已经登记的 210 张缩略图和 210 张详情图：

- 地图悬浮生态头像读取 `image_thumbnail_path`。
- 背包和队伍选择读取 `image_thumbnail_path`。
- 战斗展示读取 `image_detail_path`。

Monster Tamer 不复制维护第二套藏品图片，不把文件路径写死到战斗资料中。目录图片内容在保持模板 ID 和正式路径不变的前提下被替换后，地图、背包、队伍和战斗自动显示替换后的图片。

## 14. 明确删除的旧玩法

当前正确方案不存在以下内容：

- 公开免登录 Monster Tamer。
- 独立静态页面直接游玩。
- 仅写 `MONSTER_TAMER_DATA` 的本地单机存档。
- Monster Tamer 自有玩家怪兽。
- 初始赠送本地 Iguanignite。
- 野外怪兽捕捉。
- 捕捉球和捕捉动画。
- 本地怪兽等级和经验。
- 属性伤害克制。
- 战斗定位。
- 随机草地暗雷。
- 强制剧情和教学剧情。
- 任务列表、委托和连续 NPC 支线。
- 昼夜与天气。
- 解谜玩法。
- TMA 资产奖励。

## 15. 完整验收结果

1. 未登录、会话失效或账号门禁失败时无法进入 Monster Tamer。
2. 游戏在 FinalTMA 内以全屏覆盖层运行，关闭后返回原游戏页。
3. 背包只展示 `available_quantity > 0` 的真实 TMA 藏品。
4. 没有可用藏品时不加载地图，也不提供临时怪兽。
5. 队伍包含 1–3 个不同模板，只能在中心营地更换。
6. 进入区域和开始战斗前重新校验全部队员可用性。
7. 70 条进化链分别固定为水、火、木、风、雷之一，210 个模板都有确定战斗资料。
8. 每个模板固定 3 个技能，高阶段只强化效果。
9. 属性不产生伤害克制，只影响环境和技能附加效果。
10. 玩家和敌方怪兽全部使用 TMA 正式目录图片和资料。
11. 地图敌人可见、可绕开，不触发随机草地暗雷。
12. 五个区域、隐藏洞穴和最终守护者巢穴全部可达。
13. 开局同时开放萤光森林和潮汐湿地。
14. 五个首领依次提供藤桥生长、潮汐行走、风流滑翔、雷能充能和炽热护盾。
15. 所有能力门禁直接生效，不包含解谜流程。
16. 普通敌人和采集点在区域重入时刷新，精英、宝箱和永久门禁不重复首次结果。
17. 中心营地完全恢复；区域内恢复依赖补给、治疗果和补给点。
18. 全队倒下只返回营地，不丢失已确认探索进度，不改变 TMA 资产。
19. 击败最终守护者后记录通关并继续开放自由探索和再战。
20. 地图迷雾、队伍、区域、宝箱、精英、首领、能力和通关状态跨设备保存。
21. 任何游戏结果都不改变藏品数量、归属、预留、市场、远征、进化、分解、Mint、K-coin、Fgems、任务、邀请、VIP 或链上状态。
