## 结论

你的 TMA 盲盒项目，月卡应该定位成 **“提高留存 + 提高每日打开率 + 促进继续开盒”**，不要设计成“买了月卡就能白嫖大量高价值藏品”。首发建议只做 **一档标准月卡：499 Stars / 30 天**。后期数据稳定后，再加小月卡和高级月卡。

TMA 内数字商品和服务应使用 Telegram Stars，货币标识是 `XTR`；Telegram Bot API 也支持 Stars 订阅，订阅周期参数目前固定为 `2592000` 秒，也就是 30 天。([Telegram API][1])

## 2. 自动续费怎么做？

可以做 Stars 订阅。Telegram Bot API 的订阅发票使用 `subscription_period`，目前固定为 30 天，并且 Stars 付款的 `prices` 只能有一个价格项；官方文档还要求数字商品和服务在 Telegram 内使用 Stars。

你的业务层要额外处理：

| 模块                        | 作用                          |
| ------------------------- | --------------------------- |
| `vip_plans`               | 配置月卡档位、价格、权益                |
| `vip_subscriptions`       | 记录用户月卡状态、开始时间、结束时间、是否自动续费   |
| `vip_daily_claims`        | 记录每日福利是否领取                  |
| `vip_benefit_ledger`      | 记录 VIP 发放的积分、Fgems、道具、券     |
| `payments.star_payments`  | 记录 Stars 支付订单               |
| `payments.webhook_events` | 记录 Telegram payment webhook |
| `ops.risk_events`         | 记录退款、异常、刷奖励风险               |

发放方式建议：

```text
用户购买月卡
→ Telegram successful_payment
→ 后端记录 telegram_payment_charge_id
→ 开通 VIP 30 天
→ 每天用户手动领取福利
→ 每次领取写入 vip_daily_claims + currency_ledger
→ 到期后停止权益
→ 续费成功后延长 30 天
```

不要在购买成功后一次性发放 30 天奖励。Telegram 官方支付流程也要求在 `successful_payment` 后再交付商品，并保存 `telegram_payment_charge_id`，后续退款可能用到。
