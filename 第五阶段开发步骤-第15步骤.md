## 第 15 步：后台支付与链上运营管理

### 目标

提供最小可用后台能力，让运营能查看支付异常、webhook 事件、退款 / dispute、Mint 队列、链上交易、用户钱包，并执行安全的重试 / 暂停 / 人工处理。

### 需要先检查的现有文件 / 表

| 类型      | 路径 / 表                                                                                        |
| --------- | ------------------------------------------------------------------------------------------------ |
| Admin app | `apps/admin/package.json`                                                                        |
| Admin API | `api/admin/payments.ts`                                                                          |
| DB        | `ops.admin_users`、`admin_roles`、`admin_audit_logs`、`feature_flags`、`payments.*`、`onchain.*` |
| Server    | `api/_shared/requireSession.ts`、admin auth 需要确认                                             |

### 需要新增或修改的内容

| 文件 / 表                                            | 责任                              |
| ---------------------------------------------------- | --------------------------------- |
| `apps/admin/src/pages/PaymentsPage.tsx`（建议新增）  | 支付订单、流水、webhook、异常。   |
| `apps/admin/src/pages/MintQueuePage.tsx`（建议新增） | Mint 队列、失败、重试、人工处理。 |

| `api/admin/retry-mint.ts`（建议新增）                | 重试 Mint。                       |

| `ops.admin_audit_logs`                               | 所有操作记录。                    |

### 后端开发工作

实现 admin 鉴权。
所有后台写操作必须记录 `admin_audit_logs`。
退款、补发、重试、暂停支付、暂停 Mint 都必须有 reason。
后台不能直接改 ledger，补偿必须走 RPC。

### 数据库开发工作

确保 admin 用户和角色权限可用。
增加支付查询索引：status、created_at、user_id、invoice_payload。
增加 Mint 查询索引：status、created_at、attempt_count、next_attempt_at。
风险事件写入 `ops.risk_events`。

### 前端开发工作

后台最小页面：

- 支付订单列表。
- 支付详情。
- webhook 事件列表。
- 支付异常列表。
- dispute / refund 列表。
- Mint 队列列表。
- Mint 失败详情。
- 手动重试。
- 暂停支付 / 暂停 Mint 开关。

### 安全和幂等要求

后台 API 必须 admin 鉴权。
高风险操作二次确认。
所有后台操作写 audit。
重试操作必须幂等。
不允许后台绕过 RPC 直接发货。

### 测试要求

非管理员访问失败。
管理员查询支付。
管理员重试 failed fulfillment。
管理员重试 failed Mint。
暂停支付后用户不能新建支付。
暂停 Mint 后用户不能新建 Mint。
audit log 完整记录。

### 验收标准

- [ ] 后台可以查看支付订单。
- [ ] 后台可以查看 webhook 事件。
- [ ] 后台可以查看异常支付。
- [ ] 后台可以查看 Mint 队列。
- [ ] 后台可以重试 Mint。
- [ ] 后台可以暂停支付。
- [ ] 后台可以暂停 Mint。
- [ ] 所有后台写操作进入 audit log。
