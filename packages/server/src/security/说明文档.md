rateLimit.ts 负责频率限制，
riskControl.ts 负责把限流、用户风控标签、市场异常、支付异常、邀请异常、钱包异常、库存异常统一评估成 allow / challenge / review / deny

这两份文件可以直接服务于你的 API，例如：
const risk = createRiskControl({ supabase });

await risk.assertAllowed({
  action: 'market.buy',
  userId,
  sessionId,
  ip,
  headers: req.headers,
  idempotencyKey,
  metadata: {
    listingId,
    sellerId,
    buyerId: userId,
    priceKcoin,
    buyerBalanceKcoin,
    listingStatus: 'active',
  },
});
