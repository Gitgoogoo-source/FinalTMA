可以按下面这套流程操作。你的项目建议采用：**Supabase Storage 存图片 + 数据库存图片 URL + 前端按 URL 展示 + 图片设置 1 年缓存**。

---

# 一、最终架构

```txt
藏品图片 / 蛋图片 / 图鉴头像
↓
上传到 Supabase Storage
↓
文件名带版本号：pet001.v1.webp
↓
上传时设置 cacheControl: "31536000"
↓
数据库 items 表保存 image_url / thumbnail_url
↓
前端请求 catalog API
↓
前端用 <img src={thumbnail_url} />
↓
用户第一次打开下载图片
↓
第二次打开优先走手机 / WebView 本地缓存
```

`Cache-Control: public, max-age=31536000, immutable` 这类策略的意思是让静态资源长期缓存；`immutable` 表示资源在新鲜期内不会更新。MDN 也明确建议静态资源使用“版本号 / hash URL”，资源变化时换新 URL，这就是 cache-busting。([MDN Web Docs][1])

# 二、第一步：图片文件命名

不要这样命名：

```txt
pet001.png
pet002.png
rare_egg.png
```

推荐这样命名：

```txt
items/pet001.v1.webp
items/pet001_thumb.v1.webp

items/pet002.v1.webp
items/pet002_thumb.v1.webp

eggs/normal_egg.v1.webp
eggs/rare_egg.v1.webp

album/pet001_node.v1.webp
```

规则：

| 类型    |                推荐尺寸 | 用途         |
| ----- | ------------------: | ---------- |
| 原图    | 512×512 或 1024×1024 | 藏品详情页      |
| 缩略图   |   128×128 或 256×256 | 图鉴、背包、市场列表 |
| 蛋图片   |             512×512 | 开盒页面       |
| 图鉴节点图 |             128×128 | 图鉴连线头像     |

建议优先用 `.webp`，比 PNG/JPG 更适合移动端加载。

# 三、第二步：创建 Supabase Storage Bucket

在 Supabase 控制台：

```txt
Storage
↓
Create bucket
↓
Bucket name: game-assets
↓
Public bucket: 开启
```

建议 bucket 结构：

```txt
game-assets/
  items/
    pet001.v1.webp
    pet001_thumb.v1.webp
    pet002.v1.webp
    pet002_thumb.v1.webp

  eggs/
    normal_egg.v1.webp
    rare_egg.v1.webp
    epic_egg.v1.webp

  album/
    fire_chain_reward_box.v1.webp
```

Supabase Storage 的上传 API 支持传入文件路径，并支持 `cacheControl`、`contentType`、`upsert` 等上传选项。([Supabase][2])

# 四、第三步：上传图片时设置长缓存

写一个上传脚本，例如：

```ts
import { createClient } from "@supabase/supabase-js";
import fs from "node:fs";
import path from "node:path";

const supabaseUrl = process.env.SUPABASE_URL!;
const serviceRoleKey = process.env.SUPABASE_SERVICE_ROLE_KEY!;

const supabase = createClient(supabaseUrl, serviceRoleKey);

async function uploadAsset(localFilePath: string, storagePath: string) {
  const file = fs.readFileSync(localFilePath);

  const { data, error } = await supabase.storage
    .from("game-assets")
    .upload(storagePath, file, {
      contentType: "image/webp",
      cacheControl: "31536000",
      upsert: false
    });

  if (error) {
    throw error;
  }

  console.log("Uploaded:", data.path);
}

await uploadAsset(
  "./assets/items/pet001.v1.webp",
  "items/pet001.v1.webp"
);

await uploadAsset(
  "./assets/items/pet001_thumb.v1.webp",
  "items/pet001_thumb.v1.webp"
);
```

重点是这一段：

```ts
cacheControl: "31536000",
upsert: false
```

含义：

| 配置                          | 含义                |
| --------------------------- | ----------------- |
| `cacheControl: "31536000"`  | 让图片缓存 1 年         |
| `upsert: false`             | 不允许覆盖旧文件          |
| `contentType: "image/webp"` | 明确告诉浏览器这是 WebP 图片 |

为什么建议 `upsert: false`？

因为你不应该覆盖：

```txt
pet001.v1.webp
```

图片更新时应该新增：

```txt
pet001.v2.webp
```

然后改数据库里的 URL。

# 五、第四步：数据库保存图片 URL

你的藏品基础表可以这样设计：

```sql
alter table public.item_catalog
add column if not exists image_url text,
add column if not exists thumbnail_url text,
add column if not exists album_icon_url text,
add column if not exists asset_version int not null default 1;
```

示例数据：

```sql
update public.item_catalog
set
  image_url = 'https://你的项目.supabase.co/storage/v1/object/public/game-assets/items/pet001.v1.webp',
  thumbnail_url = 'https://你的项目.supabase.co/storage/v1/object/public/game-assets/items/pet001_thumb.v1.webp',
  album_icon_url = 'https://你的项目.supabase.co/storage/v1/object/public/game-assets/album/pet001_node.v1.webp',
  asset_version = 1
where item_code = 'pet001';
```

藏品表建议保存这些字段：

| 字段               | 作用         |
| ---------------- | ---------- |
| `image_url`      | 藏品详情大图     |
| `thumbnail_url`  | 背包、市场、图鉴小图 |
| `album_icon_url` | 图鉴节点头像     |
| `asset_version`  | 当前资源版本     |

不要把图片文件直接存在数据库里。数据库只存 URL。

# 六、第五步：后端 API 返回图片 URL

例如你的接口：

```txt
GET /api/catalog/items
```

返回：

```json
[
  {
    "item_id": 10001,
    "item_code": "pet001",
    "name": "小火龙",
    "rarity": "rare",
    "image_url": "https://xxx.supabase.co/storage/v1/object/public/game-assets/items/pet001.v1.webp",
    "thumbnail_url": "https://xxx.supabase.co/storage/v1/object/public/game-assets/items/pet001_thumb.v1.webp",
    "album_icon_url": "https://xxx.supabase.co/storage/v1/object/public/game-assets/album/pet001_node.v1.webp"
  }
]
```

前端不需要知道图片存在 Supabase、Vercel 还是 CDN。前端只需要拿到 URL，然后展示。

# 七、第六步：前端按需加载图片

React 示例：

```tsx
type ItemCardProps = {
  name: string;
  thumbnailUrl: string;
};

export function ItemCard({ name, thumbnailUrl }: ItemCardProps) {
  return (
    <div className="item-card">
      <img
        src={thumbnailUrl}
        alt={name}
        loading="lazy"
        decoding="async"
        width={128}
        height={128}
      />
      <div>{name}</div>
    </div>
  );
}
```

关键点：

```tsx
loading="lazy"
decoding="async"
```

含义：

| 配置                 | 作用          |
| ------------------ | ----------- |
| `loading="lazy"`   | 图片滚动到附近再加载  |
| `decoding="async"` | 图片解码不阻塞页面   |
| `width / height`   | 防止图片加载时页面抖动 |

图鉴弹窗里不要一次性加载 300 张大图。应该先加载缩略图，用户点击某个藏品时，再加载大图。

# 八、第七步：图片更新规则

比如你现在数据库里是：

```txt
pet001.v1.webp
```

以后你重新设计了小火龙图片，不要覆盖旧文件。

错误做法：

```txt
覆盖 pet001.v1.webp
```

正确做法：

```txt
新增 pet001.v2.webp
新增 pet001_thumb.v2.webp
```

然后更新数据库：

```sql
update public.item_catalog
set
  image_url = 'https://你的项目.supabase.co/storage/v1/object/public/game-assets/items/pet001.v2.webp',
  thumbnail_url = 'https://你的项目.supabase.co/storage/v1/object/public/game-assets/items/pet001_thumb.v2.webp',
  asset_version = 2
where item_code = 'pet001';
```

这样用户打开 TMA 后，前端拿到新 URL：

```txt
pet001.v2.webp
```

浏览器会认为这是新图片，于是重新下载。老的 `pet001.v1.webp` 不会影响新图展示。

# 九、Vercel 静态资源方案什么时候用？

小型固定 UI 资源可以放 Vercel，例如：

```txt
public/assets/logo.v1.webp
public/assets/bg_home.v1.webp
public/assets/icon_star.v1.webp
```

然后在 `vercel.json` 里设置缓存：

```json
{
  "headers": [
    {
      "source": "/assets/(.*)",
      "headers": [
        {
          "key": "Cache-Control",
          "value": "public, max-age=31536000, immutable"
        }
      ]
    }
  ]
}
```

Vercel 文档说明，静态文件会在首次请求后缓存到 Vercel 全球网络，带 hash 的静态文件在部署之间也可以继续命中缓存；浏览器侧也支持 `max-age=N, public` 和 `max-age=N, immutable` 这类缓存策略。([Vercel][3])

但你的 300 个藏品图片更建议放 Supabase Storage / CDN，而不是全部塞进前端项目。原因是：

| 方案                     | 适合内容                   |
| ---------------------- | ---------------------- |
| Vercel public 静态资源     | logo、背景、UI 图标、少量固定图    |
| Supabase Storage / CDN | 藏品图、蛋图、图鉴头像、后续会增加的运营资源 |

# 十、Vite 自动 hash 适合什么？

你的前端项目是 Vite。Vite 对被代码引用的资源会进入构建资源图，并生成 hash 文件名；这适合按钮图标、logo、少量固定 UI 图。([vitejs][4])

例如：

```tsx
import logoUrl from "@/assets/logo.webp";

export function Logo() {
  return <img src={logoUrl} alt="logo" />;
}
```

构建后可能变成：

```txt
logo.B8d91f.webp
```

但藏品图片不建议用这种方式全部 import，因为你有几百张藏品图，而且后期需要通过数据库、后台运营、活动配置来控制。

# 十一、完整推荐落地方案

你的项目可以这样定：

```txt
1. logo / 背景 / UI 小图标
   ↓
   放 Vercel public 或 src/assets
   ↓
   Vite / Vercel 处理缓存

2. 藏品图 / 蛋图 / 图鉴头像
   ↓
   放 Supabase Storage 的 game-assets bucket
   ↓
   文件名带版本号
   ↓
   上传时设置 cacheControl: 31536000
   ↓
   数据库 item_catalog 保存 URL
   ↓
   前端从 API 拿 URL 后展示
```

# 十三、最终结论

你需要操作的核心不是“把图片写死在前端”，而是：

```txt
图片上传到 Supabase Storage
↓
文件名加版本号
↓
上传时设置 1 年缓存
↓
数据库保存图片 URL
↓
前端从 API 拿 URL 展示
```

这样用户第一次打开 TMA 会下载图片；第二次打开时，只要 Telegram WebView / 手机浏览器缓存还在，并且图片 URL 没变，就会优先从本地缓存读取。缓存不是永久保险，用户清缓存、系统清理空间、Telegram WebView 缓存失效时，图片仍然会重新下载。
