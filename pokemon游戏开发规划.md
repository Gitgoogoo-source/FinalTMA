# PokePets：无尽大陆小游戏开发规划

## 1. 文档状态与唯一结论

本文是把根目录《游戏方案.md》落到当前 FinalTMA 架构上的开发规划。本文只规划，不代表功能已经实现、部署或通过真实环境验收。

本次交付固定为一套可上线运营的轻量大世界 RPG：

- 名称：`PokePets：无尽大陆`。
- 入口：现有 `/game` 页面最上方，位于藏品远征和幸运转盘之前。
- 地图：12 张地图、1 套 18×28 格碰撞模型、12 张正式背景皮肤。
- 世界结构：3 列×4 行蛇形连续大陆。
- 队伍：玩家当前可用库存中的 3 个藏品单位，同模板可重复但不能超过可用数量。
- 敌人：每张地图 3 只巡游怪物和 1 只 Boss，全部引用正式 210 藏品目录。
- 战斗：三宠按顺序出战、自动回合制、双方同时攻击、无随机数、无技能、无属性克制。
- 进度：仅保存在当前用户当前设备的浏览器 `localStorage`，不跨设备同步。
- 经济：不消耗 K-coin、Fgems、藏品、道具、TON 或 Telegram Stars，不发放任何真实奖励。
- 服务端：复用现有 `catalog.get` 与 `inventory.list` 只读接口，不新增 API、RPC、表、migration、定时任务或服务端业务写入。
- 最终事实：真实藏品目录和当前可用数量继续由服务端与数据库裁决；地图、战斗和通关进度只是本地娱乐状态，不能参与任何资产、奖励、任务、库存、交易或权限判定。

用户已经裁决地图采用四向插槽结构：同一碰撞模型预留北、东、南、西入口与出口插槽；每张地图按蛇形坐标启用对应插槽，Boss 位于本张地图有效出口前。

## 5. 用户流程

### 5.1 进入游戏中心

`/game` 固定显示：

```text
游戏中心
├── 无尽大陆
├── 藏品远征
└── 幸运转盘
```

无尽大陆卡片固定展示名称、说明、当前本机进度和一个主按钮：

- 无进度：`开始冒险`。
- 有进度：`继续冒险`，并显示当前地图与 `地图 XX / 12`。
- 本地记录损坏：`修复冒险记录`。

点击主按钮后立即打开全屏模态层。目录或库存尚未确认时先显示加载状态，不等待请求返回才提供视觉反馈。

### 5.2 队伍确认

每次从游戏中心进入冒险都先进入队伍确认：

1. 加载 `catalog.get` 与 `inventory.list`。
2. 首次进入时按 `combat_power` 从高到低自动填满三个位置；同模板数量允许占据多个位置。
3. 有存档时先恢复三个 `template_id`，再用当前 `available` 校验每个模板的使用数量。
4. 存档队伍不再满足当前可用数量时保留世界进度，清空无效队伍并显示`当前队伍已不可用，请重新选择`。
5. 玩家通过加减数量选择正好三个单位，通过上移、下移调整出战顺序。
6. 只有三个位置全部有效时，`进入无尽大陆`按钮才可提交本地状态。

队伍选择只读取库存，不创建 reservation。小游戏进行期间藏品仍可在其他设备发生真实业务变化；进入冒险和每次跨地图前重新读取库存。跨地图前发现队伍失效时暂停在队伍确认界面，已完成地图和当前地图清除结果保持不变。

### 5.3 探索循环

```text
进入当前地图
→ 四方向移动与探索
→ 碰触巡游怪物
→ 自动战斗
→ 胜利后移除该怪物
→ 三只巡游怪物全部击败
→ Boss 屏障解除
→ 击败 Boss
→ 出口或最终祭坛开放
→ 普通地图进入下一地图
→ 第 12 张地图展示大陆完成结果
```

每场战斗结束后全队恢复满血。血量不写入地图进度。

### 5.4 战败

三只玩家宠物全部倒下且敌人仍存活时：

1. 关闭战斗覆盖层。
2. 玩家回到当前地图入口出生点。
3. 全队恢复满血。
4. 已击败的其他普通怪物保持清除。
5. 本次失败怪物保持存在并回到其巡逻路线起点。
6. 页面显示`当前队伍战斗力不足`，并提供`继续探索`和`退出并调整队伍`。

### 5.5 退出与返回

- Telegram BackButton、界面`退出`按钮和原生对话框取消事件都执行同一个退出动作。
- 存档成功时立即退出到游戏中心，不弹出二次确认。
- 浏览器存储写入失败时，退出前显示`本次进度无法保存，退出后会丢失`，由用户确认退出或返回游戏。
- 退出不改变藏品、资产、任务、远征或转盘状态。

## 6. 地图与世界结构

### 6.1 固定地图模型

- 网格：18 列×28 行。
- 坐标原点：左上角 `(0, 0)`。
- 移动：仅上、下、左、右，每次一格。
- 边界：外圈默认不可通过，只开放当前配置使用的入口与出口插槽。
- 玩家长按：按下 300 毫秒后开始连续移动，之后每 160 毫秒尝试移动一格。
- 怪物巡逻：每 600 毫秒沿固定路线前进一格，到末端后循环。
- 战斗、地图过渡、队伍确认、页面隐藏时，玩家和怪物移动全部暂停。

四向插槽固定为：

| 方向 | 入口出生点 | Boss 守卫点 | 出口点     |
| ---- | ---------- | ----------- | ---------- |
| 北   | `(9, 2)`   | `(9, 5)`    | `(9, 1)`   |
| 东   | `(15, 14)` | `(13, 14)`  | `(16, 14)` |
| 南   | `(9, 25)`  | `(9, 22)`   | `(9, 26)`  |
| 西   | `(2, 14)`  | `(4, 14)`   | `(1, 14)`  |

三个巡逻路线固定为：

```text

B: (11,8) → (15,8) → (15,11) → (11,11)
```

`mapTemplate.ts` 保存唯一碰撞网格、四向插槽、三条路线和摄像机边界。12 张背景必须逐格遵守这套结构；皮肤可以把同一阻挡格画成树、冰山、岩石、岩浆或建筑，但不得改变通行结果。

## 8. 战斗规则与确定性结算

### 8.1 数值

玩家宠物和普通敌人：

```text
maxHp = 50 + combatPower
attack = floor(15 + combatPower / 3)
```

Boss：

```text
maxHp = floor((50 + combatPower) × 1.5)
attack = floor(15 + combatPower / 3)
```

战斗力只从当前已确认的目录或库存响应读取，前端不接受玩家输入的战斗力。

### 8.2 回合顺序

1. 玩家队伍按三个位置的顺序上场。
2. 每一回合玩家当前宠物和敌人同时造成各自 `attack` 伤害。
3. 每个动画回合固定 800 毫秒。
4. 敌人生命归零即为本场胜利；同回合双方归零仍判定玩家胜利。
5. 只有玩家宠物归零且敌人仍存活时，下一只玩家宠物上场。
6. 三只玩家宠物全部归零且敌人仍存活时判定战败。
7. 战斗结束后全队生命立即恢复到各自 `maxHp`。

战斗开始时由纯函数一次性生成完整回合时间线和唯一最终结果。动画只播放时间线，不在计时器回调中再次计算胜负。`跳过动画`只把播放游标移动到最后一帧，不能改变时间线或重新结算。

### 8.3 战斗覆盖层

战斗以全屏模态层内的覆盖层展示，不改变 URL：

- 敌人缩略图、名称、战斗力和 HP 条。
- 当前玩家宠物缩略图、名称、战斗力和 HP 条。
- 当前回合、队伍位置和自动战斗状态。
- `跳过动画`按钮。
- 战斗期间地图、方向键、键盘移动和巡逻计时器全部停止。

## 9. 本地进度模型

### 9.1 存储键

```text
pokepets:world-rpg:v1:<internal-user-id>
```

内部用户 ID 取自已经确认的当前应用会话。不得使用 Telegram 用户名、昵称或 `initDataUnsafe.user` 组成存储键。切换用户时只读取新用户自己的键，不能短暂展示上一用户进度。

### 9.2 最小存储结构

```ts
interface WorldRpgProgressV1 {
  schema_version: 1;
  user_id: string;
  team_template_ids: [string, string, string];
  current_stage_id: string;
  current_player_position: { x: number; y: number };
  stage_states: Array<{
    stage_id: string;
    defeated_enemy_ids: string[];
    boss_defeated: boolean;
  }>;
  completed_at: string | null;
  updated_at: string;
}
```

已完成地图、最高完成地图和已解锁地图全部由连续的 `boss_defeated` 状态推导，不重复存储。名称、图片、稀有度、战斗力、HP、攻击力和可用数量不写入存档，重新进入时从当前真实响应重建。

### 9.3 写入时机

- 队伍确认成功。
- 每次有效移动后 250 毫秒合并写入；`pagehide` 时同步冲刷最后状态。
- 普通怪物胜利。
- Boss 胜利。
- 战败回出生点。
- 地图切换完成。
- 第 12 张地图完成。
- 主动退出。

写入采用“完整对象序列化后单次 `setItem`”方式。解析时严格校验版本、用户、地图 ID、坐标范围、实例 ID、数组唯一性和三宠长度。非法记录不得部分套用。

### 9.4 重新开始

用户点击`重新开始`后先显示明确确认。确认后只删除当前用户的 world-rpg v1 键，清除队伍与全部地图进度，再进入新的队伍确认；不得清除应用会话、React Query 缓存、藏品、资产、远征或转盘状态。

## 11. API、数据库与可信边界

### 11.1 读取接口

| Route ID         | HTTP                 | 用途                                             | 调用时机                             |
| ---------------- | -------------------- | ------------------------------------------------ | ------------------------------------ |
| `catalog.get`    | `GET /api/catalog`   | 解析 12 张地图的 48 个敌人引用并校验 v1 正式目录 | 打开队伍确认时；React Query 缓存复用 |
| `inventory.list` | `GET /api/inventory` | 获取当前用户可用队伍、数量、缩略图和战斗力       | 打开队伍确认时、每次跨地图前         |

两个接口都是只读 GET，不发送幂等键，不产生 `operation_id`，不进入 operation recovery。请求仍使用现有会话、响应 Zod 校验、session generation 防迟到和 banned 门禁。

### 11.2 明确不修改的后端范围

- `packages/api-contracts` 与生成的 OpenAPI。
- `apps/api/src` 的 route handler、workflow、gateway 和环境变量。
- `supabase/migrations`、数据库 schema、表、函数、RPC、RLS、GRANT 和种子数据。
- `operations.operations`、`economy.ledger`、`inventory.holdings` 与 `inventory.reservations`。
- Vercel Functions 数量、Cron 和最大执行时间。
- Telegram webhook、Stars 支付、TON 钱包和 Mint 流程。

### 11.3 前端不可信的处理

- 玩家可以修改本机存档或前端战斗状态，但影响只限本机无奖励进度。
- 服务端不得接收 `current_stage_id`、通关状态、战斗结果、队伍战力或本地存档并据此改变任何业务事实。
- `inventory.list` 的 `available` 只用于确认进入时队伍资格；小游戏不锁定藏品，也不声称藏品正在参战。
- 任何任务、活动、奖励或排行榜都不得读取该本地进度。
- 如果本地显示与最新库存不一致，队伍资格立即按最新 `inventory.list` 回正。
- 浏览器仍不能导入 Supabase SDK、读取 service role 或直接访问数据库。

## 12. 错误、恢复与边界情况

| 场景                     | 即时反馈                                   | 确定结果                                                        |
| ------------------------ | ------------------------------------------ | --------------------------------------------------------------- |
| 正式目录加载失败         | `藏品目录加载失败`与`重新加载`             | 不进入队伍确认，不用本地敌人资料替代                            |
| 目录缺少任一配置模板     | `冒险配置加载失败`                         | 整个无尽大陆保持关闭，远征和转盘继续独立使用                    |
| 库存加载失败             | `冒险队伍加载失败`与`重新加载`             | 不使用旧可用数量开放进入                                        |
| 可用单位少于 3           | `至少需要3只可用藏品才能进入无尽大陆`      | 展示`前往开盒`和`前往藏品`，不生成试玩宠物                      |
| 存档队伍失效             | `当前队伍已不可用，请重新选择`             | 保留地图进度，重新选满三只后继续                                |
| 跨地图前队伍失效         | 暂停过渡并打开队伍确认                     | 当前地图完成状态不回退，队伍有效后进入下一地图                  |
| 本地记录 JSON 损坏       | `本地冒险记录无法读取`                     | 用户确认修复后只重置当前用户小游戏记录                          |
| 存储写入失败             | `本机无法保存冒险进度`                     | 当前前台仍可继续；退出前明确提示会丢失本次变化                  |
| 地图背景加载失败         | `地图资源加载失败`、`重新加载`、`退出冒险` | 不允许在不可见碰撞地图中移动                                    |
| 单个宠物图加载失败       | 显示`资源缺失`                             | 数值仍按确认目录结算，不用错误图片替换正式藏品                  |
| 移动目标是墙、屏障或边界 | 玩家保持原位并触发轻触觉反馈               | 不改变存档位置                                                  |
| 玩家与巡逻怪物同格       | 立即冻结地图并进入战斗                     | 同一次碰撞只创建一条本地战斗时间线                              |
| Boss 未开放时靠近        | `击败本区域全部巡游怪物后，才能挑战守卫`   | 不进入 Boss 战，不穿过屏障                                      |
| 战斗跳过动画连点         | 第一次点击立即显示既定结果                 | 不重复结算，不重复删除怪物                                      |
| 页面进入后台             | 暂停移动、巡逻和动画时钟                   | 回到前台后从同一时间线位置继续，不补算后台帧                    |
| 会话自然过期             | 由全局流程清空敏感展示并恢复一次           | 恢复成功后重读目录和库存；失败则退出冒险并要求重新进入 Mini App |
| 会话被替换或撤销         | 立即退出冒险                               | 不恢复旧会话下的 UI；本地世界记录仍按用户键隔离                 |
| 账号变为 banned          | 全局空白门禁抢占所有小游戏 UI              | 迟到资源、目录、库存和计时器结果不能恢复页面                    |
| 重新开始重复点击         | 第一次确认后按钮锁定                       | 只删除一次当前用户小游戏键                                      |
| 最终 Boss 重复进入结果   | 显示同一完成状态                           | 不产生奖励、任务或第二份完成记录                                |

## 13. UI、Telegram 与无障碍

### 13.1 全屏布局

`WorldRpgDialog` 固定使用 Telegram 稳定视口和四边内容安全区：

```text
地图名称 / 退出
世界小地图 / 地图进度 / 怪物与 Boss 状态
可裁剪地图视口与跟随玩家的纵向摄像机
三宠队伍摘要

```

地图宽度完整显示 18 列；高度超出中央视口时只移动内部摄像机，不滚动整个页面。方向键区域使用 `touch-action: none`，避免长按时触发页面滚动。

### 13.2 交互标准

- 所有触控按钮最小 44×44 CSS 像素。
- 方向键具有`向上移动`、`向下移动`、`向左移动`、`向右移动`可访问名称。
- 键盘支持方向键与 W、A、S、D；输入焦点位于按钮或对话框控件时不触发地图移动。
- 战斗 HP 条同时输出可读数值，敌人、Boss、出口和世界节点状态不能只依靠颜色。
- 打开模态层后焦点进入标题，关闭后回到无尽大陆入口按钮。
- `prefers-reduced-motion: reduce` 时取消地图滑动、浮动、缩放和粒子运动，状态变化与结果仍完整展示。
- 浅色和深色 Telegram 主题下，HUD、文本、遮罩和按钮对比度保持可读。
- 成功、失败和阻挡分别调用现有 Telegram 触觉封装，不调用 Stars、TON 或支付能力。

## 16. 一次性交付顺序

本功能不拆成最小版、演示版或延后补全项。所有工作包按依赖顺序在同一上线交付中完成：

5. 实现入口、队伍确认、地图、HUD、方向控制、巡逻、Boss、战斗、过渡、完成和错误恢复 UI。

### 17.3 数据库不变证据

## 18. 发布、回滚与完成定义

### 18.1 发布

- 只发布通过全部门禁的同一 Git commit。
- 12 张背景与代码一起进入同一 Vercel 构建，并通过 437 个正式发布资源校验。
- 不运行 migration，不清库，不修改 Supabase exposed schema，不新增环境变量。
- 不修改 Telegram webhook、Mini App URL、Stars、TON 或 Cron 配置。
- 部署后从真实开发 Bot 入口验收，普通浏览器直开不能替代 Telegram 验收。

### 18.2 回滚

- 本功能回滚只回滚 Web commit 与对应 `/assets/world-rpg/v1` 构建，不回滚数据库。
- v1 静态资源已经发布后不覆盖；修复资源使用新版本路径并与代码同 commit 切换。
- 本地存档 schema 不兼容时使用新版本存储键；不得用不完整字段猜测迁移旧记录。
- 回滚后旧本地存档可以留在浏览器中，但没有入口读取时不会影响其他功能或资产。

### 18.3 完成定义

只有同时满足以下条件才能宣布完成：

1. 产品主文档与所有架构、发布、回滚、验收资料已经同步且不存在冲突。
2. 12 张地图、48 个敌人引用、三宠自动战斗、四向蛇形连接和本地存档全部实现。
3. 目录、库存、身份、错误恢复、安全边界和资源门禁全部闭环。
4. 所有静态门禁成功，OpenAPI 与 migration 无变化。
5. 同一 commit 已部署到核对后的真实开发环境。
6. Telegram 真机与桌面验收全部通过。
7. 数据库前后证据证明无资产、库存、账本、操作或任务变化。
8. 远征、转盘及其他既有功能回归通过。

完成结果是一套可直接上线运营、但只产生本地娱乐进度的 PokePets 大世界 RPG，不是演示、mock 或临时实现。
