## 第 9 步：前端接入 TON Connect Provider 与钱包入口

### 目标

把当前钱包占位入口升级为真实 TON Connect 钱包连接入口，并在资产栏 / 钱包面板显示连接状态。

### 需要先检查的现有文件 / 表

| 类型     | 路径                                                            |
| -------- | --------------------------------------------------------------- |
| Provider | `apps/web/src/app/providers/AppProviders.tsx`                   |
| 入口     | `apps/web/src/features/assets/components/WalletEntryButton.tsx` |
| ENV      | `VITE_TONCONNECT_MANIFEST_URL`、`VITE_ENABLE_TON_CONNECT`       |
| 依赖     | `@tonconnect/ui-react`                                          |

### 需要新增或修改的内容

| 文件                                                                          | 责任                                                |
| ----------------------------------------------------------------------------- | --------------------------------------------------- |
| `apps/web/src/app/providers/TonConnectProvider.tsx`（建议新增）               | 接入 TON Connect UI provider。                      |
| `apps/web/src/features/wallet/components/WalletConnectButton.tsx`（建议新增） | 打开钱包连接。                                      |
| `apps/web/src/features/wallet/components/WalletStatusSheet.tsx`（建议新增）   | 展示地址、验证状态、断开、同步 NFT、Mint 队列入口。 |
| `WalletEntryButton.tsx`                                                       | 从占位 toast 改为打开钱包连接 / 状态面板。          |
| `public/tonconnect-manifest.json`                                             | 确认已存在；如果未发现则建议新增。                  |
| `API_ENDPOINTS`                                                               | 增加 wallet 分组。                                  |

### 后端开发工作

本步骤后端只需要提供 wallet status API 的准备。
前端拿到地址后，不能直接标记 verified，必须调用后端 proof 流程。

### 数据库开发工作

无需直接变更。
后续会写入 `core.user_wallets`、`core.wallet_proofs`。

### 前端开发工作

显示状态：

| 状态         | UI                         |
| ------------ | -------------------------- |
| 未连接       | “Connect Wallet”           |
| 连接中       | loading                    |
| 已连接未验证 | 显示短地址 + “验证钱包”    |
| 已验证       | 显示短地址 + verified 标识 |
| 验证失败     | 显示错误 + 重试            |
| 已断开       | 回到未连接                 |

### 安全和幂等要求

前端不得保存私钥。
前端只保存公开地址和 UI 状态。
真正 verified 状态来自后端。
钱包地址显示需要截断，避免 UI 泄漏完整长地址到不必要区域。

### 测试要求

未启用 feature flag 时显示暂未开放。
连接钱包成功。
用户取消连接。
连接后未验证状态。
刷新后从后端状态恢复。
断开钱包后 UI 恢复未连接。

### 验收标准

- [ ] `AppProviders` 已接入 TonConnectProvider。
- [ ] 钱包入口不再是占位 toast。
- [ ] 用户可以打开 TON Connect 钱包选择。
- [ ] 前端拿到公开地址后不会直接标记 verified。
- [ ] 刷新后可查询后端钱包状态。
- [ ] 断开钱包后 UI 状态正确。
