## 第 11 步：准备 NFT Collection、metadata 与 Mint 前置条件

### 目标

让项目具备 Mint 的前置配置：NFT Collection 地址、metadata base URI、藏品是否可 Mint、用户钱包 verified、藏品状态可 Mint。

### 需要先检查的现有文件 / 表

| 类型       | 路径 / 表                                                                                                           |
| ---------- | ------------------------------------------------------------------------------------------------------------------- |
| Contracts  | `contracts/package.json`                                                                                            |
| ENV        | `TON_COLLECTION_ADDRESS`、`NFT_METADATA_BASE_URL`、`NFT_ITEM_METADATA_BASE_URI`                                     |
| DB         | `onchain.nft_collections`、`catalog.collectible_templates`、`catalog.collectible_media`、`inventory.item_instances` |
| Validation | `CreateMintBodySchema`                                                                                              |

### 需要新增或修改的内容

| 文件 / 表                                    | 责任                                                   |
| -------------------------------------------- | ------------------------------------------------------ |
| `contracts/README.md`（建议新增）            | 记录 Collection 地址、部署网络、owner、metadata 规则。 |
| `onchain.nft_collections`                    | 写入当前网络 Collection 配置。                         |
| `catalog.collectible_templates.nft_mintable` | 确认哪些藏品可 Mint。                                  |
| `catalog.collectible_media`                  | 确认 NFT image / metadata URL。                        |
| `public/nft-metadata/` 或 Storage bucket     | 提供 Collection 和 item metadata。                     |

### 后端开发工作

读取 active collection。
校验 collection network 与用户钱包 network 一致。
生成 item metadata snapshot。
如果合约尚未开发，阶段五可以先支持“已部署 Collection 地址接入”，合约开发另设分支。

### 数据库开发工作

`onchain.nft_collections` 已有字段 `collection_address`、`metadata_url`、`content_base_url`、`royalty_config`、`status`。
需要确认至少一条 active collection。
如果还没有合约地址，Mint API 应返回“Collection 未配置”。

### 前端开发工作

藏品详情页增加 Mint 入口显示条件：

- 用户已验证钱包。
- 藏品属于用户。
- 藏品 `nft_mintable=true`。
- 藏品未挂售、未分解、未锁定。
- `nft_mint_status='not_minted'` 或 failed 可重试。

### 安全和幂等要求

metadata 生成必须后端完成。
前端不能提交任意 metadata 覆盖链上 NFT 内容。
Collection 地址必须来自数据库 / 环境，不信任前端。

### 测试要求

Collection 未配置。
Collection paused。
NFT metadata 缺失。
不可 Mint 藏品。
已 Mint 藏品重复 Mint。
用户钱包 network 不匹配。

### 验收标准

- [ ] active Collection 可查询。
- [ ] NFT metadata base URI 配置完成。
- [ ] 不可 Mint 藏品不会显示 Mint 按钮。
- [ ] 未验证钱包不能 Mint。
- [ ] Collection 未配置时给出明确提示。
- [ ] metadata 不由前端生成。

### 完成后产物

- Collection 配置记录。
- metadata 规则文档。
- 前端 Mint 入口显示规则。
- Collection 配置测试。
