## 第 14 步：实现链上 NFT 同步

### 目标

用户连接 verified 钱包后，可以手动或定时同步链上 NFT，展示钱包持有的游戏 NFT、Mint 队列和链上状态。

### 需要先检查的现有文件 / 表

| 类型       | 路径 / 表                                                                                            |
| ---------- | ---------------------------------------------------------------------------------------------------- |
| API        | `api/wallet/sync-nfts.ts`                                                                            |
| Server     | `packages/server/src/ton/nft.ts`                                                                     |
| DB         | `onchain.wallet_sync_jobs`、`onchain.wallet_nft_snapshots`、`onchain.nft_items`、`core.user_wallets` |
| Validation | `WalletNftSyncBodySchema`、`WalletNftSyncQuerySchema`                                                |

### 需要新增或修改的内容

| 文件 / 表                        | 责任                             |
| -------------------------------- | -------------------------------- |
| `api/wallet/sync-nfts.ts`        | 创建同步任务或执行轻量同步。     |
| `api/wallet/nfts.ts`（建议新增） | 查询当前钱包 NFT 快照。          |
| `onchain.wallet_sync_jobs`       | 记录同步 job、状态、错误、结果。 |
| `onchain.wallet_nft_snapshots`   | 保存链上 NFT 快照。              |
| `core.user_wallets.last_sync_at` | 同步成功后更新时间。             |

### 后端开发工作

同步流程：

1. 校验 session。
2. 校验钱包 verified。
3. 创建 `wallet_sync_jobs`。
4. 调用 TON API 查询钱包 NFT。
5. 过滤当前游戏 Collection。
6. 写入 `wallet_nft_snapshots`。
7. 更新 `onchain.nft_items.owner_address`、`last_seen_at`。
8. 更新 `core.user_wallets.last_sync_at`。
9. 返回同步数量和状态。

### 数据库开发工作

`wallet_sync_jobs` 增加 idempotency key、retry 字段。
`snapshots` 需要 `wallet_id + item_address + seen_at` 索引。
对已知 collection 的 NFT，要能关联到 `onchain.nft_items`。
如果发现链上 owner 与数据库 owner 不一致，需要记录 risk event 或 sync event。

### 前端开发工作

新增：

| 组件 / Hook         | 责任                              |
| ------------------- | --------------------------------- |
| `useSyncWalletNfts` | 触发同步。                        |
| `useWalletNfts`     | 查询链上 NFT。                    |
| `WalletSyncPanel`   | 显示同步中、成功、失败。          |
| `MintQueueSheet`    | 同时显示 Mint 队列和已 Mint NFT。 |

### 安全和幂等要求

前端不能提交 NFT 快照。
链上数据只作为外部状态，不直接改变游戏内资产归属，除非产品规则明确。
同步任务要限流。
同一钱包短时间重复同步应返回最近 job 或排队。

### 测试要求

未验证钱包同步失败。
同步空钱包。
同步包含非游戏 Collection NFT。
同步包含游戏 NFT。
TON API 失败。
重复同步。
链上 owner 变化。

### 验收标准

- [ ] 用户可以手动同步链上 NFT。
- [ ] 同步任务写入 `wallet_sync_jobs`。
- [ ] NFT 快照写入 `wallet_nft_snapshots`。
- [ ] 游戏 Collection NFT 可以关联 `onchain.nft_items`。
- [ ] 同步失败可重试。
- [ ] 前端显示同步状态。
