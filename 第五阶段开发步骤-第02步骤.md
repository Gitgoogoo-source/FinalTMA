## 第 2 步：补齐环境变量、密钥管理和 Feature Flags

### 目标

确保 Telegram Stars、Telegram webhook、TON Connect、Mint、TON API、NFT Collection、cron、admin 所需环境变量已分类配置，并明确哪些变量可进前端、哪些只能在后端。

### 需要先检查的现有文件 / 表

| 类型          | 路径 / 表             |
| ------------- | --------------------- |
| 环境变量      | `.env.example`        |
| Vercel        | `vercel.json`         |
| Feature flags | `ops.feature_flags`   |
| Settings      | `ops.system_settings` |

`.env.example` 已包含 Telegram Stars、webhook、Supabase、session、idempotency、TON、TON Connect、Mint、NFT metadata、wallet sync、feature flags 等变量。

### 需要新增或修改的内容

建议修改：

| 文件 / 表                                              | 责任                                                                                               |
| ------------------------------------------------------ | -------------------------------------------------------------------------------------------------- |
| `.env.example`                                         | 核对第五阶段所有变量是否完整；增加注释说明哪些必须在 Vercel server env。                           |
| `ops.feature_flags`                                    | 确认存在 `FEATURE_WALLET_ENABLED`、`FEATURE_TON_MINT_ENABLED`、支付暂停、Mint 暂停、链上同步暂停。 |
| `docs/security.md` 或建议新增 `docs/phase5-secrets.md` | 记录密钥管理规则。                                                                                 |

建议新增 feature flags：

| Flag                                          | 作用                         |
| --------------------------------------------- | ---------------------------- |
| `FEATURE_STARS_PAYMENT_ENABLED`               | 全局暂停 / 开启 Stars 支付。 |
| `FEATURE_PAYMENT_WEBHOOK_FULFILLMENT_ENABLED` | webhook 可落库但暂停发货。   |
| `FEATURE_WALLET_PROOF_ENABLED`                | 钱包验证开关。               |
| `FEATURE_WALLET_SYNC_ENABLED`                 | 链上 NFT 同步开关。          |
| `FEATURE_MINT_WORKER_ENABLED`                 | Mint worker 执行开关。       |
| `FEATURE_ADMIN_PAYMENT_OPS_ENABLED`           | 支付后台管理开关。           |

### 后端开发工作

后端读取环境变量时必须做启动前校验或首次调用校验。
生产环境缺少 `TELEGRAM_BOT_TOKEN`、`TELEGRAM_WEBHOOK_SECRET`、`SUPABASE_SERVICE_ROLE_KEY` / `SUPABASE_SECRET_KEY`、`APP_SESSION_SECRET` 时应拒绝支付相关操作。
`TON_MINTER_PRIVATE_KEY` 只能在 Mint worker 使用，普通 API 不应读取。
`FEATURE_TON_MINT_ENABLED=false` 时，Mint API 应返回“Mint 暂未开放”。

### 数据库开发工作

检查 `ops.feature_flags` 是否已有支付和 Mint 相关 key。
如果没有，新增 migration 插入 feature flag seed。
`ops.admin_audit_logs` 记录每一次开关变更。
`ops.system_settings` 可放置支付客服、支付过期时间、Mint retry 策略等非敏感配置。

### 前端开发工作

前端只读取 `VITE_` 公开变量。
增加前端运行时判断：如果 `VITE_ENABLE_TON_CONNECT=false`，钱包入口显示“暂未开放”。
不要让 `VITE_` 变量包含 Bot Token、service role key、TON 私钥、webhook secret。

### 安全和幂等要求

`TELEGRAM_WEBHOOK_SECRET` 必须只存在后端。
`SUPABASE_SERVICE_ROLE_KEY` 或 `SUPABASE_SECRET_KEY` 必须只存在后端。
`TON_MINTER_PRIVATE_KEY` 必须只存在 worker / server 环境。
支付开关关闭时，已支付订单仍必须允许补发，不应因为 feature flag 关闭而无法补偿。

### 测试要求

测试生产环境启用 `DEV_GACHA_PAYMENT_MODE=true` 时后端拒绝启动或拒绝执行。
测试缺失 `TELEGRAM_BOT_TOKEN` 时创建正式支付订单失败。
测试 `FEATURE_STARS_PAYMENT_ENABLED=false` 时前端无法创建新支付订单。
测试 `FEATURE_PAYMENT_WEBHOOK_FULFILLMENT_ENABLED=false` 时 webhook 可以落库但不会发货。

### 验收标准

- [ ] `.env.example` 覆盖第五阶段所有变量。
- [ ] 所有敏感变量未以 `VITE_` 开头。
- [ ] Vercel preview / production 环境变量清单已创建。
- [ ] 支付、钱包、Mint 开关可由后台或数据库控制。
- [ ] 支付暂停不影响历史订单查询。
- [ ] Mint 暂停不影响历史 Mint 状态查询。

### 完成后产物

- 更新 `.env.example`。
- 新增或更新 feature flag seed migration。
- 新增密钥管理文档。
- 新增环境变量校验测试。
