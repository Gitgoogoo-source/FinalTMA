## 第 10 步：实现 wallet challenge、ton_proof 验证、钱包保存和断开

### 目标

后端生成 challenge，前端请求钱包签名 / ton_proof，后端验证后保存公开地址，并支持断开钱包。

### 需要先检查的现有文件 / 表

| ---------- | ---------------------------------------------- |
| API        | `api/wallet/connect.ts`、`api/wallet/proof.ts` |
| Server     | `packages/server/src/ton/tonConnect.ts`        |
| Validation | `packages/validation/src/wallet.schemas.ts`    |
| DB         | `core.user_wallets`、`core.wallet_proofs`      |

| --------------------------------------- | --------------------------------------------------------------- |
| `api/wallet/challenge.ts`（建议新增）   | 创建 proof challenge。                                          |
| `api/wallet/proof.ts`                   | 验证 ton_proof，调用 RPC 保存 verified address。                |

| `core.wallet_proofs`                    | 记录 challenge、proof、status、expires_at、verified_at、error。 |

### 后端开发工作

推荐流程：

1. 前端请求 `/api/wallet/challenge`。
2. 后端创建 `core.wallet_proofs` pending 行，写入 challenge 和 expires_at。
3. 前端通过 TON Connect 请求 ton_proof。
4. 前端提交 account + proof 到 `/api/wallet/proof`。
5. 后端校验 challenge 存在、未过期、未使用。
6. 后端校验 domain、timestamp、address、public key、signature。
7. 校验通过后调用 `api.wallet_save_verified_address`。
8. `core.user_wallets.verified_at` 写入时间。
9. proof 状态更新为 verified。
10. 用户断开时更新 `core.user_wallets.status='disconnected'`、`disconnected_at`。

### 数据库开发工作

`core.wallet_proofs.challenge` 已 unique。
建议增加 `proof_hash`、`used_at`、`wallet_public_key`、`request_id`。
为 `wallet_proofs.user_id`、`wallet_id` 增加索引，performance advisor 已提示相关外键未索引。
`core.user_wallets` 当前 status 只有 `connected/disconnected/revoked`，建议：

- 方案 A：不扩展 status，用 `verified_at IS NULL` 表达 connected_unverified。
- 方案 B：扩展 status，加入 `connected_unverified`、`verified`。

推荐方案 A，迁移较小。

### 安全和幂等要求

challenge 有效期建议 5 分钟。
challenge 只能使用一次。
proof timestamp 必须校验。
domain 必须匹配当前 app 域名。
重放 proof 必须失败。
前端传来的 address 不能直接信任。
同一用户可以连接多个钱包时，必须有 primary wallet 规则；如果只允许一个主钱包，切换钱包要审计。

### 测试要求

有效 proof。
过期 proof。
重复 proof。
错误 domain。
错误 address。
错误 signature。
用户断开钱包。
同一用户重复连接同一钱包。
同一钱包被多个用户绑定时的策略测试。

### 验收标准

- [ ] 后端可以生成 wallet challenge。
- [ ] 前端可以请求 ton_proof。
- [ ] 后端可以验证 proof。
- [ ] proof 过期会失败。
- [ ] proof 重放会失败。
- [ ] 钱包公开地址保存到 `core.user_wallets`。
- [ ] `verified_at` 正确写入。
- [ ] 用户可以断开钱包。
- [ ] 前端不保存私钥。
