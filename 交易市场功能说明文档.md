# 交易市场功能说明文档

## 说明范围

这份文档只根据当前项目代码写交易市场功能，不补猜测内容，不展开接口细节，也不写没有在代码里确认到的功能。

交易市场功能主要包括这些部分：

- 用户在市场里购买别人上架的藏品。
- 用户把自己可交易的藏品上架出售。
- 用户管理自己的出售挂单。
- 用户在藏品详情页直接发起出售或取消出售。
- 后端负责校验登录、余额、归属权、挂单状态、价格、手续费和库存锁定。
- 数据库负责在一笔完整交易里完成扣款、收款、手续费记录和藏品归属转移。
- 定时任务负责刷新市场统计，并处理过期挂单。

## 一、功能定位

交易市场是项目里的用户间交易功能。当前代码里，市场只使用 K-coin 作为交易货币。

用户可以把自己拥有的、允许交易的藏品放到市场出售。其他用户可以在市场里查看这些挂单，并用 K-coin 购买。交易完成后，买家的 K-coin 会被扣除，卖家会收到扣除平台手续费后的 K-coin，藏品会转到买家名下。

前端负责展示市场页面和收集用户操作。最终能不能出售、能不能购买、价格是否有效、余额是否足够、物品是否还可卖，都会在后端和数据库里重新判断。

## 二、入口和页面

交易市场主页面有三个页签：

- 购买
- 出售
- 报价 / 管理

购买页用于浏览和购买市场挂单。

出售页用于选择自己可出售的藏品，并创建新的出售挂单。

报价 / 管理页用于查看自己的挂单统计、修改价格、取消出售。

除了交易市场主页面，藏品详情页里也有出售入口。用户可以从某个具体藏品的详情页直接发起出售，也可以取消该藏品的出售状态。

购买页顶部还会显示月卡相关入口。这个入口在购买页里出现，但它不是交易市场本身的核心交易流程。

## 三、购买功能

购买页会展示当前市场上可以买的挂单。代码里只展示仍有剩余数量，并且状态还允许购买的挂单。

用户在购买页可以做这些事情：

- 按稀有度筛选。
- 按藏品类型筛选。
- 按价格范围筛选。
- 按最新上架、价格高低、稀有度等方式排序。
- 查看每个挂单的图片、名称、编号、稀有度、类型、剩余数量和价格。
- 打开挂单详情。
- 在确认弹窗里确认购买。

挂单详情里会展示更多信息，例如：

- 出售人信息。
- 当前售价。
- 用户自己的 K-coin 余额。
- 市场参考价格。
- 最近成交情况。
- 市场最低价。
- 价格是否偏低、合理或偏高。
- 市场深度信息。

购买按钮在这些情况下会被禁用：

- 这是用户自己的挂单。
- 挂单已经售罄。
- 挂单已经不是可购买状态。
- 用户 K-coin 余额不足。

当前购买确认弹窗里，一次购买数量固定为一件。数据库里的购买逻辑也只接受单件购买。

购买成功后，前端会提示购买成功，并刷新资产和库存。后端会在同一笔数据库事务里完成扣款、给卖家入账、记录手续费、转移藏品归属、更新挂单状态和写入交易记录。

## 四、出售功能

出售页会先从后端获取当前用户可以出售的藏品。

代码里可以确认，只有满足这些条件的藏品才会出现在出售页：

- 藏品属于当前登录用户。
- 藏品当前在用户库存里是可用状态。
- 藏品没有被其他出售挂单锁住。
- 藏品没有处在 NFT 铸造排队或铸造中的状态。
- 藏品对应的配置允许交易。

出售页会把同一种藏品归在一起展示，并显示用户拥有数量和当前可出售数量。用户可以筛选和排序自己的可出售藏品，也可以看到系统给出的价格参考。

用户创建出售挂单时，可以做这些事情：

- 选择要出售的藏品。
- 选择出售数量。
- 输入单件价格。
- 查看手续费预估。
- 查看扣除手续费后预计可收到的 K-coin。
- 确认上架。

前端会检查价格是否是有效的正整数，也会阻止明显不合法的价格。最终的价格、手续费、物品归属、物品状态、是否可交易，都会以后端和数据库判断为准。

出售挂单创建成功后，被上架的具体藏品会被锁定。锁定后的藏品不会再作为普通可用库存参与其他出售流程，直到它被买走、取消出售或挂单过期。

## 五、报价 / 管理功能

报价 / 管理页用于管理当前用户自己的出售挂单。

这个页面会展示用户自己的市场统计，例如：

- 当前还在出售中的挂单数量。
- 当前还在出售中的藏品数量。
- 当前挂单总价值。
- 扣除手续费后预计可收到的 K-coin。
- 最近一天卖出的数量。
- 最近一天卖出的成交额。

用户可以在这个页面筛选自己的挂单。默认主要看出售中和部分售出的挂单。

每个自己的挂单可以做这些操作：

- 查看挂单内容。
- 修改单件价格。
- 取消出售。

修改价格时，只有挂单的出售人可以操作。已经售罄、已经取消、已经过期，或者已经不处于可出售状态的挂单，不能继续改价。

取消出售时，也只有挂单的出售人可以操作。取消成功后，未卖出的藏品会解除锁定，重新回到用户可用库存里。

## 六、藏品详情页里的出售入口

藏品详情页里也接入了交易市场出售能力。

用户在某个具体藏品的详情页里，可以直接把这个藏品上架出售。这个入口会显示手续费预估，并提醒最终手续费和到账金额以后端为准。

如果这个藏品已经处于出售状态，详情页里也可以发起取消出售。

从当前代码看，藏品详情页这个入口是围绕单个具体藏品做出售和取消出售，不是批量上架入口，也不是完整的市场管理页。

## 七、后端处理规则

交易市场相关操作都要求用户登录。

读取市场列表、挂单详情、可出售藏品、出售规则、个人挂单、个人挂单统计和市场统计，都通过后端接口完成。

创建挂单、购买、修改价格、取消出售这些会改变资产或 K-coin 的操作，会经过更严格的后端判断：

- 市场功能是否处于开启状态。
- 当前用户是否有权限操作。
- 当前用户是否触发风险检查。
- 这次操作是不是重复提交。
- 藏品是否真的属于当前用户。
- 藏品是否真的可以出售。
- 挂单是否还在有效状态。
- 买家余额是否足够。
- 买家是否正在买自己的挂单。
- 确认购买时的价格是否和当前挂单价格一致。

数据库权限也做了限制。普通前端用户不能直接改市场核心表、库存表和 K-coin 账本。涉及交易的变更必须走后端接口，再由后端调用数据库里的市场交易逻辑。

## 八、核心交易规则

当前代码里可以确认这些交易规则：

- 市场交易货币是 K-coin。
- 用户不能购买自己的挂单。
- 当前购买流程一次只买一件。
- 买家购买前会再次检查挂单价格，价格变了就不能按旧价格继续买。
- 买家余额不足时不能购买。
- 出售人只能修改或取消自己的挂单。
- 只有出售中或部分售出的挂单可以继续改价或取消。
- 已经售罄、取消、过期或暂停的挂单不能继续作为正常购买对象。
- 出售时会锁定具体藏品，避免同一件藏品被重复出售。
- 购买完成后，藏品会转到买家名下，并恢复为买家的可用库存。
- 取消出售后，未卖出的藏品会解除锁定，回到卖家的可用库存。

这些规则不是只靠前端按钮控制。数据库交易逻辑里也会重新判断，所以用户绕过页面也不能直接完成不合法交易。

## 九、价格、手续费和市场参考

市场里有手续费规则。出售页和藏品详情页会展示手续费预估，方便用户知道大概能收到多少 K-coin。

最终手续费和实际到账金额以后端计算为准。这样做是为了避免前端显示和真实交易结果不一致时，被错误结果影响账本。

市场还会维护价格参考信息。挂单详情里会展示市场最低价、近期成交、参考价格、价格健康状态和市场深度。

价格健康状态目前有这些含义：

- 偏低：价格明显低于市场参考。
- 合理：价格在参考范围内。
- 偏高：价格明显高于市场参考。
- 暂无参考：当前没有足够参考数据。

价格参考和市场深度不是用户手动填写的内容，而是由市场统计和数据库规则计算出来，用来辅助用户判断价格。

## 十、数据记录

交易市场会记录多类数据。

挂单记录用于保存谁在卖、卖什么、卖多少钱、还剩多少、当前是什么状态。

挂单物品记录用于保存一个挂单里具体锁住了哪些藏品。

订单记录用于保存购买结果。

订单物品记录用于保存这次购买具体买走了哪些藏品。

事件记录用于保存挂单创建、购买、改价、取消等变化过程。

价格快照和市场深度快照用于保存市场价格统计结果。

手续费结算记录用于保存平台手续费和卖家到账情况。

这些记录可以支撑市场列表展示、个人挂单管理、成交追踪、价格参考和后续对账。

## 十一、常见异常情况

代码里已经处理了多种市场异常情况。

用户会遇到的提示或限制包括：

- 未登录时不能使用需要身份的市场能力。
- 市场功能暂停时不能创建挂单、购买、改价或取消出售。
- K-coin 余额不足时不能购买。
- 不能购买自己的挂单。
- 挂单已售罄时不能购买。
- 挂单状态变化后不能继续按旧状态操作。
- 购买确认时价格已经变化，需要重新确认。
- 当前用户不是出售人时，不能改价或取消挂单。
- 藏品已经被锁定时，不能重复上架。
- 藏品不允许交易时，不能上架。
- 藏品处于 NFT 铸造相关处理中时，不能上架。

前端会尽量提前禁用按钮或显示提示，但真正的判断以后端和数据库为准。

## 十二、定时任务和运营支撑

当前代码里有市场相关定时任务。

一个任务用于刷新市场统计，包括价格、市场深度和价格健康状态等信息。

另一个任务用于处理过期挂单。挂单过期后，未售出的藏品需要解除锁定，回到用户可用库存。

代码里还可以看到市场结算校验相关任务，用来检查市场订单、账本和库存变更是否一致。

另外，数据库迁移里包含了运营侧使用的市场规则维护能力，例如维护手续费规则、价格健康规则和市场价格规则。这部分是后端和数据库能力。当前我没有在前端代码里确认到一个独立的市场运营后台页面，所以这里不把它写成已完成的前端页面功能。

## 十三、当前代码能确认的限制

当前代码能确认这些限制：

- 市场交易只使用 K-coin。
- 购买流程当前一次只买一件。
- 市场列表只展示还有剩余数量，并且处于可购买状态的挂单。
- 出售挂单里的藏品必须属于同一种藏品形态，不能把不同藏品混在一个挂单里出售。
- 藏品详情页的出售入口是单个藏品出售入口，不是批量出售管理页。
- 前端展示的手续费只是预估，最终以后端计算为准。
- 我没有在当前代码里确认到独立的市场运营后台页面。
