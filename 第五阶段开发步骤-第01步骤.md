## 第 1 步：冻结当前第四阶段基线

### 目标

建立第五阶段开发前的稳定基线，确保任务、邀请、签到、分红、开盒开发模式、交易、库存、图鉴现有功能不被破坏。

### 需要先检查的现有文件 / 表

| 类型 | 路径 / 表                                                                                                                 |
| ---- | ------------------------------------------------------------------------------------------------------------------------- |
| 前端 | `apps/web/src/features/tasks/pages/TasksPage.tsx`、`apps/web/src/features/box/pages/BoxPage.tsx`                          |
| API  | `api/boxes/create-open-order.ts`、`api/boxes/result.ts`                                                                   |
| DB   | `gacha.draw_orders`、`gacha.draw_results`、`inventory.item_instances`、`economy.currency_ledger`、`tasks.*`、`album.*`    |
| RPC  | `api.gacha_create_order`、`api.gacha_process_dev_paid_order`、`api.gacha_get_draw_result`、`api.task_*`、`api.referral_*` |
| 测试 | `tests/api/`、`supabase/tests/`                                                                                           |
| 配置 | `.env.example`、`vercel.json`                                                                                             |

### 需要新增或修改的内容

不新增业务能力。只新增第五阶段基线说明文档和发布分支策略。

建议新增：

| 建议新增                           | 责任                                                  |
| ---------------------------------- | ----------------------------------------------------- |
| `docs/phase5-baseline.md`          | 记录当前可用功能、不可用功能、测试命令、冻结 commit。 |
| Git 分支 `phase5-payments-onchain` | 第五阶段开发主分支。                                  |

### 后端开发工作

确认 `DEV_GACHA_PAYMENT_MODE` 在开发和测试环境可用，生产环境必须关闭。
确认 `api/boxes/create-open-order.ts` 在 dev mode 下仍能跑通抽卡闭环。
确认 `api/boxes/result.ts` 可查询历史订单结果。
确认 `requireSession` 对所有需要登录的 API 生效。

### 数据库开发工作

记录当前 Supabase migration 最新版本。当前最新已确认到 `20260527045136 phase4_redact_commission_source_ids`。
导出现有 schema 差异说明。
记录 `payments` 和 `onchain` 表当前 rows 状态，避免上线时把 dev 数据误认为正式支付数据。

### 前端开发工作

确认现有底部导航、资产栏、开盒页、任务页正常。
确认 `PaymentPendingSheet` 当前只作为等待占位，不代表正式支付完成。

### 安全和幂等要求

不得在第五阶段分支中删除现有 idempotency 逻辑。
不得把 `.env`、Bot Token、service role key、TON 私钥提交到仓库。
不得修改现有 `currency_ledger` 历史数据。

### 测试要求

运行现有 API test。
运行现有 DB test。
手动验证 dev 支付模式开盒不会破坏任务分红逻辑。
确认任务领取、签到、邀请分享仍可用。

### 验收标准

- [ ] 第四阶段任务中心功能仍可访问。
- [ ] dev mode 开盒仍可生成 `draw_results`。
- [ ] `currency_ledger` 与 `user_balances` 可对账。
- [ ] 创建第五阶段开发分支。
- [ ] 记录当前 migration 最新版本。
- [ ] 生产环境 `DEV_GACHA_PAYMENT_MODE=false` 的检查项已写入上线 checklist。

### 完成后产物

- 新增基线文档。
- 新增第五阶段开发分支。
- 一次完整回归测试记录。
- 当前 Supabase migration 和 API 状态快照。
